# Release Notes: v0.1.0

**Release Date:** 2025-01-15

This is the initial release of PresentationSpec, a Go-first specification library for creating data-driven presentations.

## Highlights

- **Go-first design**: Go types are the source of truth, JSON Schema is generated automatically
- **Multiple output formats**: HTML and PDF export out of the box
- **Rich widget library**: 12 widget types for dashboards and executive presentations
- **Live development**: Dev server with hot reload for rapid iteration
- **Data integration**: Load data from files and APIs directly into presentations

## Features

### Core Specification

- **PresentationSpec type** with metadata, theme, narrative, slides, data sources, and export configuration
- **12 widget types**:
  - `metric_card` - KPI metrics with trends and sparklines
  - `content_block` - Markdown/HTML content
  - `chart` - Bar, line, pie, area, and more via Chart.js
  - `table` - Data tables with configurable columns
  - `risk_list` - Risk items with severity and mitigation
  - `decision_list` - Decisions requiring action
  - `checklist` - Task lists with completion status
  - `diagram` - Mermaid, PlantUML, Graphviz, D2 diagrams
  - `callout` - Info, warning, error, success, tip callouts
  - `quote` - Quotations with attribution
  - `image` - Images with captions
  - `code` - Code blocks with syntax highlighting

- **6 slide types**: title, content, dashboard, section, summary, blank
- **Layout system** with predefined templates and custom regions
- **Theme support** with colors, typography, density, and aspect ratio

### Schema Generation

- JSON Schema generated from Go types using `invopop/jsonschema`
- Embedded schema for runtime validation
- CLI tool (`genschema`) for schema regeneration

### HTML Rendering

- Static HTML output via `a-h/templ` templates
- Responsive CSS with CSS custom properties
- Keyboard navigation (arrow keys, space, home/end)
- Hash-based slide linking

### PDF Export

- PDF generation via headless Chrome (`chromedp`)
- Configurable page size (Letter, A4, Legal, Tabloid)
- Landscape/portrait orientation
- Background graphics support

### Development Server

- Live reload on file changes
- Automatic rebuild with error reporting
- Configurable port and host
- `render serve` command

### Watch Mode

- File watching with configurable interval
- Auto-rebuild on spec changes
- `render --watch` flag

### Data Sources

- **JSON file loading** - Load data from local JSON files
- **CSV file loading** - Parse CSV with headers as keys
- **REST API loading** - Fetch data from HTTP endpoints
- **Authentication support**:
  - Bearer token
  - Basic auth
  - API key (custom header)

### Syntax Highlighting

- Prism.js integration via CDN
- Auto-detection of language from `language` attribute
- Tomorrow theme (dark)

### Diagram Support

- Mermaid.js integration via CDN
- Support for flowcharts, sequence diagrams, class diagrams, etc.

### Chart Support

- Chart.js integration via CDN
- Chart types: bar, line, pie, donut, area, scatter, radar
- Configurable options (legend, grid, axis labels)
- SVG fallback for basic charts

### Speaker Notes View

- Separate speaker.html with presenter view
- Current slide and next slide preview
- Speaker notes panel
- Presentation timer (start/pause/reset)
- Keyboard navigation
- `render --speaker-notes` flag

### Validation

- JSON Schema validation via `xeipuuv/gojsonschema`
- `render --validate` flag for pre-render validation
- `render --validate-only` flag for validation without rendering

## CLI Commands

```bash
# Render to HTML
render presentation.json -o output/

# Render to PDF
render presentation.json --format pdf -o presentation.pdf

# Validate spec
render presentation.json --validate-only

# Watch mode
render presentation.json -o output/ --watch

# Dev server
render serve presentation.json --port 8080

# Generate speaker notes
render presentation.json -o output/ --speaker-notes

# Generate schema
genschema -o schema/presentation.schema.json
```

## Dependencies

- `github.com/a-h/templ` - HTML templating
- `github.com/spf13/cobra` - CLI framework
- `github.com/invopop/jsonschema` - JSON Schema generation
- `github.com/xeipuuv/gojsonschema` - JSON Schema validation
- `github.com/chromedp/chromedp` - Headless Chrome for PDF

## Test Coverage

- 33 unit tests across packages
- Tests for data loading, rendering, validation, and spec types

## Known Limitations

- Database data sources not yet implemented
- PPTX export not yet available
- No custom theme file loading (use inline theme in spec)
- Mermaid/Chart.js require internet connection (CDN)

## Upgrade Notes

This is the initial release. No upgrade path required.
